## Overview

This guide covers the complete deployment of the Golden Team NEO Enterprise Platform on a self-hosted VPS. The stack includes 10 integrated services managed by Docker Compose, with PostgreSQL 16 as the primary database, Redis 7.2 for caching and queuing, and Nginx as the SSL-terminating reverse proxy.

**Estimated deployment time:** 2–4 hours for a first-time deployment on a fresh server.

## 2. Prerequisites Installation

Run these commands on your fresh Ubuntu 22.04 server as root or with sudo:

```bash
# Update system
apt update && apt upgrade -y

# Install Docker
curl -fsSL https://get.docker.com | sh
systemctl enable docker
systemctl start docker

# Install Docker Compose plugin
apt install -y docker-compose-plugin

# Install Certbot for SSL
apt install -y certbot

# Install AWS CLI (for S3 backups)
apt install -y awscli

# Install useful tools
apt install -y git curl wget htop ufw fail2ban

# Configure firewall
ufw allow 22/tcp    # SSH
ufw allow 80/tcp    # HTTP (redirect to HTTPS)
ufw allow 443/tcp   # HTTPS
ufw enable

# Configure fail2ban (brute force protection)
systemctl enable fail2ban
systemctl start fail2ban
```

## 4. Deployment Steps

> **Version Note:** This package uses **Odoo 19.0** (released September 2025, patch 19.1 January 2026). The Docker image `odoo:19.0` is updated every 2–3 weeks by the Odoo team and was last pushed on March 5, 2026.

### Step 1 — Clone the deployment package

```bash
# Upload the gt-neo-deploy folder to your server, or clone from your private repo
scp -r gt-neo-deploy/ user@your-server-ip:/opt/gt-neo-deploy
cd /opt/gt-neo-deploy
```

### Step 2 — Configure environment

```bash
cp .env.example .env
nano .env   # Edit all required values
```

**Required values to set in .env:**
- All `DOMAIN_*` variables — replace `yourdomain.com` with your actual domain
- `POSTGRES_SUPERUSER_PASSWORD` — strong random password
- `SMTP_HOST`, `SMTP_USER`, `SMTP_PASSWORD` — your email provider credentials
- `MANUS_API_KEY` — your Manus API key
- `OPENAI_API_KEY` — your OpenAI API key

All other passwords will be auto-generated by the init script if left as `CHANGE_ME_*`.

### Step 3 — Run initialization

```bash
chmod +x scripts/init.sh scripts/backup.sh
bash scripts/init.sh
```

The script will:
1. Validate all prerequisites
2. Generate any missing secure passwords
3. Start PostgreSQL, Redis, and MinIO
4. Initialize all databases and schemas
5. Start all application services
6. Request SSL certificates via Let's Encrypt
7. Start Nginx
8. Print all access URLs

### Step 4 — Initial application setup

After deployment, complete these one-time setup steps:

**Odoo ERP (v19.0):**
```
1. Navigate to https://erp.yourdomain.com
2. Create database: odoo_production
3. Install modules: Accounting, CRM, Purchase, Inventory, Project
4. Install OCA modules from /mnt/extra-addons
5. Configure Chart of Accounts for Saudi Arabia
6. Set company name: Golden Team Trading Services
7. Enable Odoo 19 AI Assistant features (Settings → Technical → AI)
```

**OpenProject (ASTRA PM):**
```
1. Navigate to https://pm.yourdomain.com
2. Complete setup wizard
3. Create project: ASTRA-001 (main construction project)
4. Configure work package types and statuses
5. Add team members
```

**OrangeHRM:**
```
1. Navigate to https://hr.yourdomain.com
2. Complete installation wizard (use existing PostgreSQL credentials)
3. Configure company structure and departments
4. Add all employees
5. Configure leave types and approval workflows
```

**Metabase:**
```
1. Navigate to https://bi.yourdomain.com
2. Complete setup wizard
3. Add data source: PostgreSQL (host: postgres, db: neo_platform)
4. Add data source: PostgreSQL (host: postgres, db: odoo_production)
5. Import dashboard templates from docs/metabase-dashboards/
```

**Rocket.Chat:**
```
1. Navigate to https://chat.yourdomain.com
2. Complete setup wizard
3. Create channels: #general, #projects, #finance, #hr, #announcements
4. Configure NEO bot integration (webhook URL from NEO portal settings)
```

**Outline:**
```

3. Create initial collections: Policies, Procedures, Projects, HR
4. Import existing SOPs and documentation
```

## 6. Backup Configuration

### Automated daily backups

```bash
# Add to crontab (runs at 2:00 AM daily)
crontab -e
# Add this line:
0 2 * * * /opt/gt-neo-deploy/scripts/backup.sh >> /var/log/gt-neo-backup.log 2>&1
```

### Manual backup

```bash
bash /opt/gt-neo-deploy/scripts/backup.sh
```

### Restore from backup

```bash
# Restore PostgreSQL
gunzip -c /var/backups/gt-neo/postgres_full_YYYYMMDD_HHMMSS.sql.gz | \
  docker exec -i gt-postgres psql -U gtadmin

# Restore specific database
gunzip -c /var/backups/gt-neo/db_neo_platform_YYYYMMDD_HHMMSS.sql.gz | \
  docker exec -i gt-postgres psql -U gtadmin neo_platform
```

docker compose logs -f odoo

# Restart a service
docker compose restart odoo

## 8. Redis Database Reference

| Database | Purpose | Access Pattern |
|:---|:---|:---|
| DB 0 | API Response Cache | `redis-cli -n 0` |
| DB 1 | BullMQ Job Queues | Managed by BullMQ |
| DB 2 | Session Management | `redis-cli -n 2` |
| DB 3 | Pub/Sub (transient) | Subscribe/Publish only |
| DB 4 | Rate Limiting | `redis-cli -n 4` |

```bash
# Connect to Redis CLI
docker exec -it gt-redis redis-cli -a YOUR_REDIS_PASSWORD

# Check queue depths
docker exec -it gt-redis redis-cli -a YOUR_REDIS_PASSWORD -n 1 KEYS "bull:*"

# Check cache hit rate
docker exec -it gt-redis redis-cli -a YOUR_REDIS_PASSWORD INFO stats | grep keyspace
```

## 10. Security Checklist

Before going live, verify all items in this checklist:

- [ ] All default passwords changed in .env
- [ ] Firewall configured (only ports 22, 80, 443 open)
- [ ] fail2ban installed and running
- [ ] SSL certificates installed and auto-renewing
- [ ] Odoo `list_db = False` in odoo.conf
- [ ] PostgreSQL not exposed on public network interface
- [ ] Redis not exposed on public network interface
- [ ] Backup script running and tested
- [ ] MFA enabled for all admin accounts
- [ ] Audit log integrity check scheduled
- [ ] SMTP configured for approval notifications
